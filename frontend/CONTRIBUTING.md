# Contributing to Stay Rooted

Thank you for your interest in contributing to Stay Rooted! We're excited to work with you.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature/fix
4. Make your changes
5. Test thoroughly
6. Submit a pull request

## Code Standards

- Write clean, readable TypeScript
- Follow the existing code style
- Add comments for complex logic
- Use meaningful variable and function names
- Keep components small and focused

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add prayer sharing feature`
- `fix: Resolve location permission issue`
- `docs: Update README with setup instructions`
- `style: Format code with Prettier`

## Pull Request Process

1. Update README.md with details of changes if needed
2. Update type definitions if adding new features
3. Ensure all TypeScript errors are resolved
4. Test on both iOS and Android if possible
5. Request review from maintainers

## Code Review

All submissions require review before merging. We'll provide feedback and work with you to ensure quality.

## Questions?

Open an issue or reach out to the maintainers.
