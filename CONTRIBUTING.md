# Contributing to Stay Rooted

Thank you for your interest in contributing to Stay Rooted!

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a new branch for your feature
4. Make your changes
5. Run tests and linting
6. Submit a pull request

## Development Setup

See README.md for complete setup instructions.

## Code Style

We use ESLint to maintain code quality. Run `npm run lint` before committing.

## Testing

Run tests with `npm test`. All new features should include tests.

## Pull Request Process

1. Update documentation for any new features
2. Ensure all tests pass
3. Update the README.md if needed
4. Follow the existing code style

## Code of Conduct

Be respectful and constructive in all interactions.
